# Contributing to AURA

Thanks for contributing to AURA.

---

## Development Principles

- Reality over hype  
- No fake capability claims  
- Keep docs aligned with code  
- Prefer stable fixes over flashy features  
- UI must not overclaim backend truth  

---

## Setup

1. Clone the repository  
2. Create and activate a Python virtual environment  
3. Install dependencies from `requirements.txt`  
4. Configure `.env`  
5. Run AURA locally  
6. Run tests before making changes  

---

## Branch Naming

- fix/...  
- feat/...  
- refactor/...  
- docs/...  
- test/...  

---

## Pull Request Rules

- Keep changes focused  
- Explain what changed and why  
- List files touched  
- Mention risks or limitations  
- Update docs when behavior changes  
- Add or update tests if logic changes  

---

## Quality Standard

Before opening a PR:

- Code runs  
- Tests pass  
- Behavior is honest  
- No fake features  
- Documentation is updated
