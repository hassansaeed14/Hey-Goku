# Security Policy

## Reporting a vulnerability

Do NOT open public issues for security problems.

Report privately with:
- affected area
- steps to reproduce
- impact
- possible fix (if known)

---

## Sensitive Areas

- authentication
- session management
- PIN system
- permission system
- memory access
- execution paths
- private data

---

## Principle

AURA is private-first.

Security must be real, not cosmetic.
